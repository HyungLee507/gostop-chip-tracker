# 고스톱 칩 트래커 요구사항

## 목표
친구들끼리 3~4명이 고스톱을 칠 때, 점수 계산은 사람이 직접 하고,
웹 화면에서는 **플레이어 간 칩(코인) 이동 결과만 기록/표시**하는 아주 단순한 도구를 만든다.

## 핵심 방향
- 백엔드 없이 **프론트엔드만**으로 구현
- **React 기반**으로 구현
- 복잡한 룰 계산 없음
- 숫자 위주로 단순하게 표시
- 빠르게 입력하고 바로 현재 칩 현황을 볼 수 있어야 함

## 사용자 시나리오
1. 사용자가 게임을 시작한다.
2. 플레이어 3~4명의 이름을 입력한다.
3. 각 플레이어는 동일한 시작 칩 수를 가진다.
4. 한 판이 끝날 때마다,
   - 누가
   - 누구에게
   - 몇 칩을 줬는지
   기록한다.
5. 화면에는 각 플레이어의 현재 칩 잔액이 즉시 반영된다.
6. 최근 칩 이동 내역도 함께 볼 수 있다.

## 필수 기능

### 1. 게임 초기 설정
- 플레이어 수: 3~4명
- 플레이어 이름 입력 가능
- 모든 플레이어의 시작 칩 수는 동일
- 시작 칩 수 직접 입력 가능

### 2. 칩 이동 기록
- 아래 3가지를 입력할 수 있어야 함
  - 보내는 사람
  - 받는 사람
  - 칩 수
- 기록 버튼을 누르면 즉시 반영
- 점수 계산/룰 계산은 하지 않음

### 3. 현재 칩 현황 표시
- 각 플레이어의 현재 보유 칩 표시
- 숫자가 한눈에 잘 보여야 함
- 누가 얼마를 가지고 있는지 바로 보여야 함

### 4. 이동 내역 표시
- 최근 거래 내역 표시
- 예시: `철수 -> 영희 : 3`
- 최신 내역이 위로 오면 좋음

## 있으면 좋은 기능
- 거래 1건 삭제
- 전체 초기화
- 브라우저 새로고침 후에도 데이터 유지(localStorage)
- 모바일 화면 대응

## 비기능 요구사항
- UI는 화려할 필요 없음
- 숫자가 잘 보이는 단순한 화면이면 됨
- 입력 속도가 중요함
- 백엔드/로그인/서버/DB 불필요

## 추천 화면 구성

### 화면 1: 메인 단일 화면
한 페이지 안에 아래가 모두 있으면 됨.

#### A. 게임 설정 영역
- 플레이어 이름 3~4명
- 시작 칩 수 입력

#### B. 칩 이동 입력 영역
- 보내는 사람 선택
- 받는 사람 선택
- 칩 수 입력
- 기록 버튼

#### C. 현재 현황 영역
- 플레이어별 현재 칩 수 표시

#### D. 최근 이동 내역 영역
- 최근 기록 리스트 표시

## 데이터 모델 예시
```ts
Player {
  id: string
  name: string
}

Transfer {
  id: string
  fromPlayerId: string
  toPlayerId: string
  amount: number
  createdAt: number
}

GameState {
  players: Player[]
  initialChips: number
  transfers: Transfer[]
}
```

## 계산 규칙
- 모든 플레이어의 시작 칩은 동일
- 현재 칩 = 시작 칩 + 받은 칩 합 - 준 칩 합
- 전체 칩 합은 항상 동일해야 함

## 제외 범위
아래는 이번 버전에서 하지 않음.
- 고스톱 점수 계산
- 룰 자동 판정
- 승패 계산
- 서버 저장
- 로그인
- 멀티 사용자 실시간 동기화

## 개발 메모
- Vite + React 정도로 가볍게 시작 가능
- 상태는 useState/useMemo 정도면 충분
- 저장은 localStorage로 처리 가능
- 디자인보다 동작 우선

## Codex에게 바로 넘길 수 있는 한 줄 요약
`React 프론트엔드만으로, 3~4명 플레이어가 동일 시작 칩을 가지고 시작해서 매 판마다 플레이어 간 칩 이동만 기록하고 현재 잔액/최근 내역을 보여주는 단일 페이지 웹앱을 만들어줘. 점수 계산 로직과 백엔드는 필요 없다.`
